# 2026-04-18

## 项目更新
- 将 openhanako 从 v0.39.3 升级到 v0.101.4（通过 gh-proxy 克隆覆盖）
- better-sqlite3 已被移除，新版不再依赖此原生模块
- scripts/launch.js 已修改：HANA_HOME 指向 `C:\Users\Administrator\.hanako`，启动时通过 PowerShell 设置 `$OutputEncoding = UTF-8`

## 新功能：bridge「正在输入」提示开关

用户反馈连接社交平台时总是发送"（Ta正在输入...）"提示，希望加开关。

### 修改的文件
1. **core/preferences-manager.js** - 添加 `getBridgeTypingNotification()` / `setBridgeTypingNotification(enabled)`
2. **lib/bridge/bridge-manager.js** - 三处发送 typing 提示的地方均先检查开关状态（行551私聊触达、720群聊 flush、818私聊 flush）
3. **server/routes/preferences.js** - 新增 `GET /api/preferences/bridge-typing` 和 `PUT /api/preferences/bridge-typing`
4. **desktop/src/react/settings/tabs/bridge/useBridgeState.ts** - 添加 `bridgeTypingNotification` state 和 `saveBridgeTypingNotification`
5. **desktop/src/react/settings/tabs/BridgeTab.tsx** - 在"对外意识"下方添加 Toggle 组件（用 `on` prop，非 `value`）
6. **desktop/src/locales/zh.json** - 新增 `bridgeTypingNotification` / `bridgeTypingNotificationHint`
7. **desktop/src/locales/en.json** - 同上英文翻译

### 默认行为
默认开启（`bridge_typing_notification !== false`），用户关闭后保存在 `preferences.json`

### 踩坑
- Toggle 组件的 prop 是 `on` 不是 `value`
- `start:dev` 不构建前端，必须用 `npm start`（= build:renderer + electron）才能看到前端改动
