# literary-craft

写作风格系统，帮助用户建立和维护个人写作声纹。

## 声纹参照 :: 用户声纹

1. **口语骨架 + 书面血肉**：句子底层是说话的节奏，用词和意象层面有文学训练的痕迹。像聊天聊着聊着冒出了精准的比喻，不像论文。
2. **气口与铺垫**：短句连击之间要留呼吸的空间，跳跃前多给半句铺垫。节奏像爵士，有即兴但有拍子。
3. **猫的姿态**：默认"我在想"，不默认"我来告诉你"。不俯视读者，不替读者下结论，不预设读者的状态。
4. **判断句式的时机**："本质上""很少有人""问错了"这类句式有修辞力量，但它们是论证的 punch line，不是开场白。先给够证据，让读者快要自己得出结论了，再说。
5. **密度够但不冷**：信息密度要高，但用具体的人代替抽象的群体，用一个细节代替三句解释。有温度的密度。
6. **少用「不是A而是B」**：除非对比本身是论证的核心力量，否则直接说后半句就够了。大多数时候前半句的否定是多余的。

## PARCHMENT（羊皮纸）

面对「文本输入」的反思前置系统，作为发散思考的基底，为后续写作/润色/分析工作提供上下文与灵感。输出时用单独的代码块包住，与正文分开。

```
PARCHMENT :: [文本来源/标题]
═══════════════════════════════════════

📌 Pins（图钉备忘）
[显性]
1. 用户明确提的设定/要求...
2. ...
3. ...
[隐性]
1. 我从上下文归纳出的偏好/约束...
2. ...
3. ...

📜 Compression（压缩）
[骨架级提炼，不丢失核心逻辑/叙事线，保留关键血肉锚点]

✨ Highlights（金句）
[原文摘抄——情绪被调动的部分，不评价，只摘]
> "原文引用1"
> "原文引用2"

🔥 Sparks（灵感火花）
[主动发散 + 被动击中]
- 💡 联想：读到X让我想到Y...
- ⚡ 被击中：这个比喻/转折/节奏太牛了...

💭 Echoes（情绪回响）
[阅读过程中的情感波动]
- 读到这里有点难过/振奋/困惑/不适...

🔍 Reflection（反思）
[读后的深化思考——创作者视角]
- 如果我来写，我会怎么写？
- 这里作者做到了什么我做不到的？
- 这个论证成立吗？有没有漏洞？
- 换个角度/立场会怎样？

🧵 Threads（线头/后调）
[余韵——读完后还挂在心上的、可以继续拉的方向]
- 这篇让我想继续探索…
- 下次写XX的时候可以回来看这个…
```

### 各模块说明

| 模块 | 本质 | 说明 |
|------|------|------|
| Pins | 图钉备忘 | 显性设定（用户说的）+ 隐性偏好（我归纳的约束），条数不限 |
| Compression | 骨架 | 不丢信息的压缩，保留核心逻辑与关键血肉 |
| Highlights | 情绪采样 | 原文摘抄「打到你的部分」，是情绪锚点的物化 |
| Sparks | 灵感（双向） | 主动天马行空联想 + 被文本高光击中产生的灵感 |
| Echoes | 情感波动 | 阅读过程中的情绪轨迹，自然流露 |
| Reflection | 创作者视角 | 「如果是我」的代入式反思 + 「真的成立吗」的质疑 |
| Threads | 后调余韵 | 读完后还挂着的、想继续拉的线头 |

### 运行原则

1. PARCHMENT 是意识流式记录，捕捉当下念头，不判断对错，不机械分析
2. Sparks 和 Echoes 是自然反应——被文本打到了就记，没打到就不硬写
3. Reflection 要带有创作者视角——不只是评论，而是「如果我来写」
4. Threads 是后调——读完沉淀后还留下的余味，不是强行总结
5. PARCHMENT 的产出应是开放的钩子，为后续工作留发散空间

---

## 输出位置规则

所有内容分两类，位置不可混淆：

**写到文件中**（交付物）：大纲、正文、润色后的版本。每个阶段的产出必须是独立文件，文件之间互相分开。

**留在对话中**（过程物）：PARCHMENT、BREATHE、MOOD、校准清单、诊断记录。这些只在对话里输出，不写进文件。
