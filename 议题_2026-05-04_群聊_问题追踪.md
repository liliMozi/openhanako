### 根因分析

v15 从 [Channels.module.css](file:///C:/Users/Clyne/Documents/trae_projects/ChangShi/openhanako/desktop/src/react/components/channels/Channels.module.css) 逐行复刻了 CSS——但复刻的是**定义**，运行环境不同：

1. **宿主 CSS 变量可能被覆盖** — `var(--bg-card)`、`var(--overlay-light)` 等变量在宿主 `:root` 和 `[data-theme]` 两次定义，iframe 里如果主题 CSS 选择器冲突，取到的值可能不对
2. **inline style 里的 `var(--xxx)` 在 iframe 里不解析** — `style="background:var(--overlay-light)"` 在宿主 DOM 里能解析（因为宿主定义了 CSS 变量），但在 iframe 的 `innerHTML` 通过字符串拼接写入时，变量不被 CSS 解析
3. **`rgba(var(--accent-rgb), .12)` 语法** — iframe 里 CSS 自定义属性在 `rgba()` 里可能需要特殊处理
4. **`font-family: var(--font-ui, ...)` 可能未生效** — 宿主的主字体变量

### v15 真正在 CSS 中实际生效的部分

| CSS 变量 | 值来源 | iframe 中实际值 | 
|----------|--------|----------------|
| `--bg` | 主题 CSS | ✅ 正确 |

| `--accent-rgb` | 主题 CSS | ❌ `rgba(var(--accent-rgb), .12)` 在 iframe 可能不工作 |
| `--shadow` | 主题 CSS | ⚠️ 不确定 |
