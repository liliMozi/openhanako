## better-sqlite3 ABI 不匹配修复
- Electron 38 内嵌 Node.js v22 (ABI 139)，但 better-sqlite3 是用 Node.js v20 (ABI 131) 编译的
- 执行 `npx electron-rebuild -f -w better-sqlite3` 解决
- 同步复制到打包 app 的 `node_modules/better-sqlite3/build/Release/`

## 功能：微信 ClawBot 接入
- **需求**：集成微信 ClawBot 插件，让用户可以通过微信与 Hanako 对话
- **技术方案**：直接实现微信 ClawBot 的 ilink HTTP JSON API 协议（长轮询 getUpdates + sendMessage），不依赖 openclaw 框架
- **认证流程**：扫码登录 → 获取 bot_token → 保存到 preferences → 自动启动长轮询
- **新增文件**：
  - `lib/bridge/wechat-adapter.js`：微信 ClawBot 适配器（含 createWechatAdapter 工厂函数 + startWechatLogin/pollWechatLogin 登录流程）
- **修改文件**：
  - `lib/bridge/bridge-manager.js`：ADAPTER_REGISTRY 注册 wechat 平台
  - `lib/bridge/session-key.js`：SESSION_PREFIX_MAP 注册 `wx_dm_` 前缀
  - `server/routes/bridge.js`：status 返回微信实例状态 + 新增 `/api/bridge/wechat-login-start` 和 `/api/bridge/wechat-login-poll` 端点
  - `desktop/src/react/settings/tabs/BridgeTab.tsx`：新增微信 ClawBot 配置 UI（扫码登录二维码、状态显示、owner 选择）
  - `desktop/src/styles.css`：新增 `.bridge-wechat-*` 系列样式
- **协议细节**：
  - API base URL: `https://ilinkai.weixin.qq.com`
  - Headers: `AuthorizationType: ilink_bot_token`, `Authorization: Bearer <token>`, `X-WECHAT-UIN: random`
  - 端点: `ilink/bot/getupdates`, `ilink/bot/sendmessage`, `ilink/bot/get_bot_qrcode`, `ilink/bot/get_qrcode_status`
  - 消息类型: message_type=1(用户), message_type=2(Bot)
  - sessionKey 格式: `wx_dm_{userId}`
  - session expired errcode: -14，暂停 1 小时
  - contextToken 需要在每次回复时回传
- **npm 包**：安装了 `@tencent-weixin/openclaw-weixin@1.0.3`（仅作为参考，适配器直接用 fetch 实现协议）

## 微信二维码 CSP 修复 (2026-03-24)
- 微信 API `qrcode_img_content` 返回的是网页 URL 而非图片 URL
- 安装了 `qrcode@1` 库，在服务端用 `QRCode.toDataURL()` 直接生成二维码 PNG data URI
- CSP 中已添加 `https://liteapp.weixin.qq.com`（但最终不再依赖外部 URL）

## 微信消息不显示在 sidebar 的修复 (2026-03-24)
- **问题 1**：`/api/sessions` 和 `/api/bridge/sessions` 只读取当前活跃 agent 的 `bridge-sessions.json`，如果切换了 agent 就看不到其他 agent 的 bridge sessions
  - **修复**：改为遍历所有 agents 的 bridge-sessions.json，跨 agent 合并 bridge sessions
  - 修改文件：`server/routes/sessions.js`、`server/routes/bridge.js`
- **问题 2**：收到 `bridge_message` WS 事件时前端没有刷新侧边栏 session 列表
  - **修复**：在 `app-ws-shim.ts` 的 `bridge_message` handler 中添加 `_sb().loadSessions()` 调用
  - 修改文件：`desktop/src/react/shims/app-ws-shim.ts`
